CHAPTER 2

Colours are numbers too (in disguise)

'#1FA8C9' looks like a secret code. It is - but a simple one. A colour is three numbers: how much RED, GREEN and BLUE light to mix, each from 0 to 255, written in hexadecimal (base 16). #1FA8C9 = red 31, green 168, blue 201 - mostly green + blue = teal.

REAL GAME CODE src/core/config.ts:10
carColor: '#1FA8C9', // try '#E8402A' (lava red) or '#8A2BE2' (purple)
PLAYGROUND · PAINT SHOP
Boss move: rgb(255, 0, 255) - full red + full blue = magenta. Zero everything = black. Max everything = white. You now know how every screen on Earth works.
CHAPTER 3

true / false, and the mighty IF

Some knobs aren't numbers - they're switches: true or false (a boolean). And the most important word in all of programming is if: do this, but only when that.

Real example: when you hit a tree, the game decides crunch-or-smash with an if:

REAL GAME CODE (the knob it reads) src/core/config.ts:34

CHAPTER 8

TypeScript: code with seatbelts

Sundown Run isn't plain JavaScript - it's TypeScript. Same language, one addition: you can write down what TYPE of thing each value is (: number, : string, : boolean), and the computer checks your work before the game even runs. It's spell-check for code.

REAL GAME CODE src/core/store.ts:4
export type CarBodyId = 'coupe' | 'striker' | 'muscle' | 'wedge'

That line says a car body can ONLY be one of those four words. Type 'van' anywhere and TypeScript underlines it in red before you've even saved. You be the checker - would TypeScript allow these?

const topSpeed: number = 'really fast'
✗ Error. topSpeed promises to be a number, and 'really fast' is a string (words in quotes). TypeScript catches it instantly - in plain JavaScript this bug would hide until the speedo said "NaN".
const body: CarBodyId = 'van'
✗ Error. The type only allows the four real bodies. This is why the garage can never show a car that doesn't exist - the mistake is impossible to write.
let laps: number = 3
laps = laps + 1
✓ Allowed. number + number is a number - promise kept. (let means it can change; const means it never will.)
function boost(power: number) { ... }
boost()
✗ Error. The machine needs a power number fed in, and it got nothing. TypeScript refuses to even start the game until you feed it.
This is why the game can be 15,000 lines and still work: hundreds of tiny promises, all checked automatically, all the time. When you edit the real code in VS Code, red underlines = TypeScript talking to you. It's on your side.
